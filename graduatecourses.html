Graduate Courses

I found that my graduate school courses were somewhat different than the undergraduate computer science courses that I took. The graduate school textbooks had more of an emphasis on mathematics. We were more likely to use a textbook that taught the mathematical concept first before introducing the concept in a programming language or pseudocode. Another difference that I found was that there was a greater emphasis on working in teams. Additionally, we were often given topics and had to search through the literature then present our findings.

Object-Oriented Software Development

We used the Eclipse IDE to program Java using JUnit for test-driven development. The book used in the course was Applying UML and Patterns by Craig Larman. The main project for the course involved following the process outlined in the book to design a Java program. There was a strong emphasis on Agile and teamwork.

System Architecture

We studied various software architectures while comparing and contrasting them. The book used in the course was Software Architecture by Taylor, Medvidovic and Dashofy. This course involved a lot of communicating what we learned through presentations.

Distributed Systems

This course was split between theory of distributed systems using research projects and practical application using REST architecture. The main project was implementing a gradebook created with the Netbeans IDE using JAX-RS to implement the APIs.

Analysis of Algorithms

The course involved implementing and analyzing algorithms that solve specified problems. The final project consisted of implementing a graph algorithm to determine the quickest route between cities while accounting for available fuel.

Project Management

In this course, we improved our understanding of project managing by selecting a project to research and create deliverables for throughout the project's life cycle.

Application of Artificial Intelligence/Machine Learning

This course was a broad survey of artificial intelligence and machine learning with a mix of theory and practical application. Among the more practical applications covered were the use of AIML to program a bot and the use of Knime for supervised and unsupervised learning. The most interesting project from this course was programming the strategy for a computer player using Java and having the my team's computer player compete against other teams in the game Reversi.

Issues in Database Management

This course covered relational datababases from a mathematical perspective, using relational algebra, as well as from a practical perspective, using SQL. Databses were covered from both a theoretical and a practical perspective. The course covered NOSQL, including MongoDB.

Verification and Testing

This course covered software testing with an emphasis on independent research and writing. In the project for this course, I used Ruby to create a program while using RSpec to create the tests before coding.

Capstone

We applied finance and accounting in the context of software using the book Return on Software by Steve Tockey as a guide. We studied ethics as it relates to software engineering.
